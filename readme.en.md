# Mellitus.IA

> Intelligent medical assistant: scan medical records, track patient progress and receive clinical insights that correlate all exams - revealing patterns that would otherwise go unnoticed.

Full-stack application in **Next.js**  that converts medical reports (PDF, photos or manual input) into longitudinal clinical monitoring, with automated report reading and AI-driven analysis.

---

## Features

-  **Authentication** with  email and password (NextAuth + bcrypt), with 30-day session persistence.
-  **patient management**  isolated per user - each physician can only access their own patients (identified by CPF).
-  **AI-powered report reading** - Send a PDF or a photo, and the values will be automatically exctracted for analysis.
-  **Manual Entry** - 14 medical report categories with ~60 parameters and pre-mapped units
-  **editing and removal of exams** - Correct or remove previously registered exams at any time.
- **patient detail page** - basic data, most recent measurements, a mini-dashboard, and a full history of all values (with highlights for out-of-range results).
- **Diabetes risk using Machine Learning** - a Random Forest model (trained in JavaScript, without Python/ONNX) classifies Type 2 diabetes risk as LOW/MEDIUM/HIGH.
- **AI-driven clinical analysis** - correlates the entire history, validates the ML model’s risk classification, and identifies trends, cross-domain correlations, alerts, and recommendations.
- Instant alerts (non-AI) - out-of-range values are flagged immediately based on clinical rules.


## Stack


| Layer | Technology |

### Prerequisites
- Node.js 20+
- A PostgreSQL database (e.g., Azure, Supabase, or local)
- An Anthropic API key (for report reading and analysis)

### 1. Install dependencies
```bash
npm install
```

### 2. Environment Variables
Create a `.env` file in the root directory with:
```env
DATABASE_URL="postgresql://user:password@host:5432/database?sslmode=require"
NEXTAUTH_SECRET="a-random-secret-string"
NEXTAUTH_URL="http://localhost:3000"
ANTHROPIC_API_KEY="sk-ant-..."
```

> Special characters in the password (e.g., `@`) must use URL encoding (`%40`). In the Vercel dashboard, provide values without quotes.

> For local development with PostgreSQL installed on your machine, use:
> DATABASE_URL="postgresql://postgres:password@localhost:5432/mellitus-ia"

### 3. Database
npx prisma db push      # creates tables from the schema
npx prisma db seed      # (optional) populates 9 demo patients

### 4. Run
npm run dev

Access http://localhost:3000.

> Demo patients are associated with the first user in the database. Sign up and run the seed, or log in with the account that owns the data.

## Risk Model (Machine Learning)

Type 2 diabetes risk is classified by a Random Forest model trained in JavaScript (ml-random-forest), without Python or ONNX dependencies. The model comes pre-trained in src/lib/risk_model.json and is loaded at runtime by src/lib/riskModel.js.

To retrain using the Kaggle dataset "Diabetes Prediction Dataset" (Mohammed Mustafa):

1. Download the CSV and save it as model/diabetes_prediction_dataset.csv.
2. Run:
npm run train

This generates a new src/lib/risk_model.json with metrics (accuracy, AUC, recall, precision, F1) printed to the console.

## Deployment (Vercel)

1. Import the repository into Vercel.
2. Configure the environment variables: DATABASE_URL, NEXTAUTH_SECRET, NEXTAUTH_URL (with the production URL), and ANTHROPIC_API_KEY — without quotes.
3. The build runs prisma generate && next build.
4. Allow database firewall access for Vercel connections.
